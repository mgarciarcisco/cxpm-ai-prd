Design Notes - Option A: Side Navigation

Key Features

  • Two-column layout: Fixed 250px sidebar with scrollable section navigation + main content area
  • Sticky sidebar navigation: Shows all 9 requirement sections with item counts; highlights active section in teal
  • Sticky header: Breadcrumbs, page title with total count badge (24 items), "In Progress" status, and action buttons
  • Meetings panel: Horizontal scrollable cards showing 3 meetings with status badges (Applied/Ready to Apply)
  • Collapsible sections: Expand/collapse with chevron rotation; section icon, title, count badge, and "Add" button
  • Always-visible actions: Edit and delete buttons shown on every item (not hover-only for better discoverability)
  • Sticky footer: "Mark as Reviewed" primary action button fixed at bottom

Design Tokens Used

  • --teal-accent: #4ECDC4 - Primary accent color for active states, buttons, bullets
  • --bg-page: #f8fafc - Page background
  • --bg-card: #ffffff - Card/section backgrounds
  • --border-color: #e5e7eb - Default borders
  • System fonts: -apple-system, BlinkMacSystemFont, 'Segoe UI', Roboto, sans-serif

Interaction Patterns

  • Clicking sidebar nav items scrolls main content to that section and highlights the nav item
  • Section headers are clickable to expand/collapse (chevron animates 90deg)
  • "Add" buttons on section headers open a modal pre-filled with that section
  • Edit button enables inline editing (shown in full implementation)
  • Delete button shows confirmation dialog
  • Meeting cards with "Ready to Apply" status show "View Items" link

Accessibility Considerations

  • All interactive elements have visible focus states (teal outline)
  • aria-expanded on section headers indicates collapsed/expanded state
  • Semantic HTML: <nav>, <main>, <header>, <footer>
  • Action buttons have descriptive title attributes
  • Sufficient color contrast for all text

Responsive Notes

  • Below 768px: Consider collapsing sidebar into a hamburger menu or bottom tab bar
  • Meetings panel scrolls horizontally on all screen sizes
  • Sticky footer ensures primary action is always accessible
