Design Notes - Option C: Horizontal Tabs with Table View

  • Layout: Sticky header with breadcrumbs, title, count, status badge, and action buttons. Horizontal tabs below for section navigation.
  • Tabs: Each section has its own tab with icon, name, and count badge. Active tab has teal underline. Tabs scroll horizontally on small screens.
  • Table view: Requirements displayed in a clean table format with columns for checkbox, row number, requirement text, source, and actions.
  • Source chips: Clickable chips showing which meeting the requirement came from. "Manual" entries have distinct yellow styling.
  • Actions on hover: Edit and delete buttons appear when hovering over a row, keeping the UI clean.
  • Bulk selection: Checkboxes for selecting multiple requirements at once. Selected count shown in footer.
  • Meetings drawer: Slide-out panel on the right showing all meetings. Triggered by "Meetings" toggle button in filter bar.
  • Sticky footer: Pagination info and "Mark as Reviewed" button always visible at bottom of screen.
  • Benefits: Table view allows quick scanning of many requirements. Tabs reduce scrolling. Drawer keeps meetings accessible without taking up screen space.
  • Accessibility: Proper ARIA labels, keyboard navigation support, and focus states throughout.
  • Responsive: Tabs scroll horizontally on mobile. Filter bar stacks vertically. Footer adapts to narrow screens.
