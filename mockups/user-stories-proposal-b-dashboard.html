Tips for Best Results
  • Focus on user value and benefits, not implementation
  • Keep stories small, independent, and deliverable
  • Include specific, testable acceptance criteria
  • Describe the "who," "what," and "why" clearly
Expected Format
Your requirements should describe: • The user persona or role • The specific action or capability needed • The business value or benefit • Clear acceptance criteria • Any relevant context or constraints The AI will structure these into proper JIRA User Stories.
